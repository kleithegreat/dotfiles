# desktopctl Architecture

## Scope

Current implementation state for the unified `desktopctl` binary and its repo
integration.

## Current Crate Layout

| Piece | Current implementation |
| --- | --- |
| Crate manifest and packaging | `desktopctl/Cargo.toml` defines the binary crate; `desktopctl/default.nix`, `overlays/local-packages.nix`, and the `packages.x86_64-linux` export in `flake.nix` package it as a flake-exposed Nix derivation. |
| CLI dispatch | `desktopctl/src/main.rs` defines the full clap tree for `daemon`, `theme`, `brightness`, `hypr`, `launch-quickshell`, `portal`, `night-light`, and `sun`, including `hypr lid-switch`, the nested `hypr input status/set`, `hypr animations save/clear`, and `hypr keybinds save/clear` surfaces, the hyphen-prefixed value handling needed by `theme set ... -1` and `hypr input set sensitivity -0.1`, and the live Rust dispatch. |
| Shared path helpers | `desktopctl/src/paths.rs` resolves the repo root from `DESKTOPCTL_REPO`, legacy `desktopctl_REPO`, or `~/repos/dotfiles`, exposes `repo_path()` for repo-relative helper lookups, provides shared XDG home/runtime fallbacks, and exposes the shared `desktopctl.db` path. |
| Theme schema and validation | `desktopctl/src/theme/schema.rs` and `desktopctl/src/theme/resolve.rs` define the `ColorScheme` and `ThemeState` contract, including required scheme `appearance`, centralized app-theme metadata including KTextEditor theme names, compiled default theme-state values, canonical field ordering, per-target system-font and mono-font offsets, color-scheme loading, `theme_state` persistence with legacy `themes/state.json` import support, and additive-schema backfill for older persisted state rows that are missing newer required keys. The default-state logic in `desktopctl/src/theme/schema.rs` now derives the default `dark_hint` from the default color scheme's declared appearance. |
| Theme JSON compatibility | `desktopctl/src/theme/json.rs` implements Python-style JSON rendering, including preserved object order, 2-space indentation, compact object spacing, and `ensure_ascii=True` escaping used by generated JSON targets and Quickshell-facing CLI output. |
| Theme CLI surface | `desktopctl/src/theme/mod.rs` implements the current `desktopctl theme` command surface, including `all`, `sync`, scoped apply commands, `set`, `preset`, preset-file management, and `status` / `list-*` inspection output. The `colors` and `fonts` CLI paths now derive from the registry's declared target `state_keys` instead of carrying duplicate target lists locally. The `list-schemes --json` path emits richer preview objects for Quickshell, including identity, appearance, named colors, bright variants, and the 16-color terminal palette, while `list-wallpapers --json` now emits filename-ordered wallpaper entries with cached preview-image paths for Quickshell. `set_dark_hint()`, `cmd_set()`, and `cmd_preset()` still keep `dark_hint` on the direct theming path, while the theme tests there now cover preserving an explicit `dark_hint` across `color_scheme` changes. |
| Wallpaper preview cache | `desktopctl/src/theme/wallpaper_browser.rs` filters supported wallpaper files, builds cache-keyed preview images under `$XDG_CACHE_HOME/desktopctl/wallpaper-previews/`, tolerates per-file decode failures without dropping the rest of the listing, and renders the JSON payload consumed by Quickshell's wallpaper pane. |
| Theme orchestration | `desktopctl/src/theme/orchestrator.rs` owns generated-file headers, assembly strategies, sync-safe filtering, dependency selection, ordered target application, atomic file replacement, concat merges, repo-relative base-path resolution, post-write hooks, and best-effort runtime reloads. Per-key fanout plus the `theme colors` / `theme fonts` apply scopes are now derived from each target's declared `TargetMetadata.state_keys`, with the existing `filter_wallpaper = false` exception still removing `wallpaper` from the `color_scheme` fanout. |
| Theme target registry | `desktopctl/src/theme/targets/mod.rs` replaces Python auto-discovery with a typed registry and hand-registered target set for all 27 current theme targets, with each target now declaring its consumed `ThemeState` keys in metadata. |
| Theme targets | File-writing targets live under `desktopctl/src/theme/targets/*.rs`; `desktopctl/src/theme/targets/bat.rs`, `desktopctl/src/theme/targets/snappy_switcher.rs`, `desktopctl/src/theme/targets/vicinae.rs`, `desktopctl/src/theme/targets/vscode.rs`, and `desktopctl/src/theme/targets/qt.rs` read per-scheme app metadata from `ColorScheme`. The command-only `desktopctl/src/theme/targets/chromium.rs` target patches active Chromium profile prefs for web fonts, `dark_hint`-driven browser chrome, and managed page-size cleanup; `desktopctl/src/theme/targets/helium.rs` reuses the same Chromium-family preference patching for Helium's `~/.config/net.imput.helium/` user data directory. The `qt` target writes KDE icon-theme state into `kdeglobals`, syncs Kate/KWrite to scheme-declared KTextEditor theme names, and resolves a same-family light/dark scheme from `dark_hint` before falling back to the selected scheme. The `ghostty`, `vicinae`, concat, `zsh`, and `where_is_my_sddm_theme` target ownership details remain as documented in the theming architecture. Other notable runtime-heavy ports include the `cursor`, `gtk`, `gtksourceview`, `quickshell`, and `wallpaper` targets. |
| Hyprland helpers | `desktopctl/src/hypr.rs` now covers window helpers, live `hyprctl keyword` calls, the laptop lid-switch monitor policy, the generated `input-runtime.conf`, `animations-override.conf`, and `keybinds-override.conf` files, plus the save/clear flows that rewrite those managed override files and reload Hyprland. `hypr lid-switch closed` queries `hyprctl monitors -j` and disables the internal monitor only when a different output is active; `open` restores the internal monitor with the provided monitor spec; `sync` reads `/proc/acpi/button/lid/*/state` and only applies the closed-lid action. The same file also carries focused tests for the parser, generated file format, managed-input value parsing helpers, animation/keybind override rendering, lid-switch state parsing, and socket lookup, which still prefers the current `HYPRLAND_INSTANCE_SIGNATURE`, falls back to `/tmp/hypr/<sig>/.socket2.sock`, and otherwise picks the newest discovered socket under the runtime or `/tmp/hypr` trees. |
| Focus tracker | `desktopctl/src/daemon/focus.rs` implements the live focus subsystem: one-second SQLite accumulation, migration from the legacy `focustime.db`, desktop-entry resolution, Hyprland socket listening, and atomic JSON summary writes. |
| Shared solar logic | `desktopctl/src/solar.rs` resolves cached or GeoClue coordinates, computes sunrise/sunset with the NOAA-derived port, derives the separate local-clock `dark_hint` window, and exposes `sun status` plus next-event selection. |
| Night-light CLI and helpers | `desktopctl/src/night_light.rs` adds the daemon-backed `desktopctl night-light {status,on,off,auto,toggle}` surface, the Unix-socket client helpers with finite read/write timeouts, fallback status reporting, and `hyprsunset` process / IPC helpers. When `hyprsunset` is already running, temperature reads and writes now go through its `~/.hyprsunset.sock` IPC endpoint so manual temperature changes can update in place without tearing the filter down first. Permission errors on the daemon socket are surfaced instead of treated as local fallback. |
| Daemon supervisor | `desktopctl/src/daemon/mod.rs` starts the focus tracker, solar scheduler, and Unix-socket server under one tokio runtime, sharing one night-light controller between the solar and socket tasks and coordinating clean shutdown on `SIGTERM` / `SIGINT`. |
| Night-light controller | `desktopctl/src/daemon/night_light.rs` stores the live `auto` / `on` / `off` mode, keeps the in-session manual temperature, derives the effective `hyprsunset` state from solar status, and remains the only live writer of `hyprsunset`. It validates manual temperatures before mutating in-memory mode. The first solar-status update marks the current scheduled `dark_hint` value for one reconciliation so daemon startup repairs stale persisted hints; later solar-status updates mark both 23:00 dark-on and 06:00 dark-off schedule edges. Reconcile then applies the pending `dark_hint` value once through the helpers in `desktopctl/src/night_light.rs`, which delegate to `set_dark_hint()` in `desktopctl/src/theme/mod.rs`, without coupling that write to the current night-light mode. A failed dark-hint apply leaves the pending target intact for a later reconcile. The helper layer now prefers in-place `hyprsunset` IPC temperature updates before falling back to a process restart. |
| Solar scheduler | `desktopctl/src/daemon/solar.rs` replaces the old systemd timer script chain with an in-process scheduler that recomputes solar status immediately, sleeps until the next sunrise / sunset / 23:00 / 06:00 event or 2-hour repair tick, and asks the shared controller to reconcile the effective mode. |
| Socket server | `desktopctl/src/daemon/server.rs` serves newline-delimited JSON requests on `$XDG_RUNTIME_DIR/desktopctl.sock`, including the daemon-owned `night_light.status`, `night_light.set`, and `night_light.toggle` methods. It refuses to remove a stale path unless it is actually a Unix socket, only unlinks an existing socket after stale-socket connect failures such as refused/reset connections, surfaces permission and other probe errors, and runs blocking controller work on Tokio's blocking pool instead of the async worker tasks. |
| Existing helper ports | `desktopctl/src/brightness.rs`, `desktopctl/src/launch.rs`, and `desktopctl/src/portal.rs` remain the active ports for brightness status/setting/stepping/dimming, Quickshell launch env export, and the directory picker helper. The brightness helpers now prefer sorted `/sys/class/backlight` names plus `brightnessctl` for implicit step/dim/set commands, fall back to DDC/CI VCP `0x10` through `ddcutil`, enumerate DDC/CI displays with `ddcutil detect --brief` for JSON status, expose a Quickshell-facing `devices` array containing every readable backlight/DDC device, store DDC dim state under `$XDG_RUNTIME_DIR/desktopctl/`, notify Quickshell through `qs ipc`, and accept oversized `brightness set` percentages so the backend clamp contract is honored. `desktopctl/src/portal.rs` now gives `dbus-monitor` a bounded startup window and correlates portal responses to the `OpenFile` request handle before accepting a returned directory. |

## Repo Integration

Nix/Home Manager ownership for package export, installation, and activation is
documented in `docs/nix/ARCHITECTURE.md`.

| Surface | Current implementation |
| --- | --- |
| Package and activation surface | `overlays/local-packages.nix` exports the packaged binary, `home/packages.nix` installs it, and the `home.activation.applyTheme` hook in `home/default.nix` bootstraps the mutable Hyprland fragments before running `desktopctl theme sync`. |
| Quickshell integration | `config/quickshell/popups/SettingsPopup.qml`, `config/quickshell/DisplayService.qml`, and `config/quickshell/HyprlandConfigService.qml` use the JSON and command surfaces for theme state, wallpaper previews, night-light control, and Hyprland settings, while `config/quickshell/shell.qml` exposes the shell-level `theme.apply` IPC bridge with argv-safe tokenization. |
| Hyprland integration | `config/hypr/autostart.conf` launches `desktopctl daemon`, launches Quickshell through `desktopctl launch-quickshell`, and reapplies the persisted wallpaper after `awww-daemon` starts; `hosts/laptop/autostart.conf` runs `desktopctl hypr lid-switch sync --internal eDP-1`, `hosts/laptop/input-devices.conf` binds lid switch events to `desktopctl hypr lid-switch`, and `config/hypr/keybinds.conf` / `config/hypr/hypridle.conf` use the `hypr`, `brightness`, and `night-light` subcommands directly. |

## Migration Status

- The tracked Python theming entry point and session-script surfaces are gone:
  `themes/apply-theme`, `scripts/focus-daemon.py`, `scripts/sun-schedule`,
  `scripts/brightness-step.sh`, `scripts/dim-screen.sh`,
  `scripts/toggle-float.sh`, `scripts/launch-quickshell.sh`,
  `config/quickshell/scripts/dir-picker.py`, and `home/sun-schedule.nix` are
  no longer part of the repo.
- A `themes/lib/` directory can still reappear locally if stale Python bytecode
  artifacts are generated by an old checkout or tool cache, but it is no longer
  a tracked implementation surface.

## Verification

The Rust test suite covers the stable contracts that are practical to unit-test:
theme schema/backfill/target rendering, target fanout, Quickshell-facing JSON
shapes, Hyprland override rendering, solar schedule derivation, night-light
socket envelopes and controller state, brightness conversion/parsing helpers,
portal response parsing, launch environment resolution, and shared path helpers.

Some live orchestration remains integration-bound because it shells out to
session services: `hyprctl`, `hyprsunset`, `brightnessctl`, `ddcutil`,
`busctl`, `dbus-monitor`, and desktop app config reload hooks. Keep new logic in
small pure helpers where possible before adding more subprocess choreography.
